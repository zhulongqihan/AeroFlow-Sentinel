# 退改签处理延迟处置手册

## 风险事件名称
- 事件名: `RefundProcessingDelay`
- 级别: 高
- 触发条件: 退改签处理队列积压连续 10 分钟超过阈值

## 业务影响
- 旅客退款到账超时，投诉量增加
- 改签确认时间变长，影响出行体验
- 客服人工介入量上升
- 清结算和对账周期被拉长

## 常见根因
1. refund-service 消费能力下降，MQ backlog 持续升高
2. 航司直连接口响应慢，退款确认状态长时间不回传
3. 退改签规则计算复杂，审核链路耗时增加
4. 多币种退款和汇率换算服务阻塞

## 建议排查路径
1. 查询 Prometheus 风险事件，确认队列积压和处理时延的变化趋势
2. 查询 payment-settlement 日志，确认退款请求是否集中超时
3. 查询 ancillary-service 与 booking-transaction 日志，检查改签附加服务是否联动失败
4. 检查最近是否调整了退改签规则、退款审批策略或航司接口路由

## 即时处置建议
1. 提升退款消费组并发，优先处理即将超 SLA 的请求
2. 对慢航司或慢渠道启用异步确认与人工审核兜底
3. 暂时关闭高成本规则校验，优先恢复主流程吞吐
4. 对大额或国际票退款启用单独隔离队列

## 恢复判定
1. 队列积压恢复到阈值以下
2. 平均退款处理时长回到基线水平
3. 改签确认失败率和退款超 SLA 比例明显下降

## 复盘关注点
1. MQ backlog 监控和自动扩容策略是否完善
2. 航司接口慢响应是否有灰度隔离机制
3. 退款和改签链路是否需要拆分独立容量池
