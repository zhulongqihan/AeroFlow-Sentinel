# 出票失败率上升处置手册

## 风险事件名称
- 事件名: `TicketingFailureRateRise`
- 级别: 严重
- 触发条件: 出票失败率连续 5 分钟高于 5%

## 业务影响
- 出现已扣款未出票，人工补单压力增大
- 旅客无法及时收到票号，改签和登机前服务受阻
- 退款和客诉量快速上升
- 履约成功率下降，渠道赔付风险升高

## 常见根因
1. ticketing-service 调用 BSP 或航司出票接口超时
2. PNR 创建成功但票号回写失败，状态机不一致
3. 支付成功回调堆积，补偿任务执行延迟
4. 座位锁定过期导致出票前校验失败

## 建议排查路径
1. 查询 Prometheus 风险事件，确认出票失败率抬升时间和范围
2. 查询 booking-transaction 日志，确认 PNR 创建、座位锁定、票号回写状态
3. 查询 payment-settlement 日志，检查支付回调和结算链路是否存在超时
4. 检查最近是否进行了出票路由策略、供应商优先级或补偿任务配置变更

## 即时处置建议
1. 启用人工出票兜底或将流量切换至备用出票供应商
2. 暂时提高补偿任务优先级，优先处理已支付订单
3. 对已扣款未出票订单启动专项巡检并限制继续放量
4. 对高失败航线实施保护策略，减少异常扩散

## 恢复判定
1. 出票成功率恢复到 97% 以上
2. 已支付待出票订单积压恢复到正常范围
3. 票号回写失败和补偿重试日志明显下降

## 复盘关注点
1. 状态机是否支持支付成功和出票成功解耦补偿
2. 是否建立供应商级别的失败熔断和快速切换能力
3. 是否需要对高价值订单提供更强的一致性保障
