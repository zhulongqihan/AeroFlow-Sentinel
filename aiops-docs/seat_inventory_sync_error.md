# 座位库存同步异常处置手册

## 风险事件名称
- 事件名: `SeatInventorySyncError`
- 级别: 中高
- 触发条件: 座位库存同步失败率连续 5 分钟超过 3%

## 业务影响
- 用户选座失败、附加服务下单失败
- 出现座位已售但页面仍可选的风险
- 改签时舱位展示不准确，影响转化和客诉

## 常见根因
1. 座位图缓存与实际库存刷新延迟
2. ancillary-service 与航司座位接口状态不一致
3. MQ 同步消息丢失或重复消费
4. 热门航班座位锁定冲突严重

## 建议排查路径
1. 查询 ancillary-service 日志，确认座位图刷新和库存回写情况
2. 查询 booking-transaction 日志，检查座位锁定与释放是否正常
3. 核对缓存版本、航司库存版本和本地库存版本
4. 检查最近是否发布了选座规则、舱位映射或库存同步任务变更

## 即时处置建议
1. 对异常航班关闭在线选座，切换人工或值机选座兜底
2. 缩短缓存 TTL，强制刷新热门航班座位图
3. 对库存同步消息启用补偿扫描任务
4. 对高冲突航班限制并发选座流量

## 恢复判定
1. 座位同步失败率恢复到 1% 以下
2. 座位图与实际库存一致性恢复正常
3. 选座失败和重复售卖告警不再持续出现

## 复盘关注点
1. 座位库存版本管理是否足够清晰
2. 热门航班是否需要专门的缓存和限流策略
3. 座位锁定与释放链路是否具备幂等补偿能力
