# GDS/供应商接口超时处置手册

## 风险事件名称
- 事件名: `GdsSupplierTimeout`
- 级别: 高
- 触发条件: 单一供应商接口超时率连续 5 分钟超过 10%

## 业务影响
- 搜索结果缺失、报价失真或出票失败
- 单供应商依赖过重时，热门航线整体可用性下降
- 回退到备用供应商后成本可能显著上升

## 常见根因
1. 供应商侧限流或网络抖动
2. 连接池耗尽，重试风暴导致雪崩
3. TLS 证书或鉴权 token 异常
4. 路由层未及时摘除异常供应商节点

## 建议排查路径
1. 查询 gds-gateway 日志，确认 provider、timeout、http_status 分布
2. 对比不同供应商成功率，判断是否为单点故障还是整体回源异常
3. 检查连接池、线程池、重试次数和熔断状态
4. 检查最近是否更新了鉴权配置、证书或路由规则

## 即时处置建议
1. 降低异常供应商流量权重或直接摘除
2. 打开熔断并限制重试次数，避免放大故障
3. 启用缓存结果和备用供应商兜底
4. 对关键航线启用静态推荐结果，优先保证搜索可用性

## 恢复判定
1. 供应商超时率恢复到基线
2. 搜索结果完整率和报价成功率恢复正常
3. 熔断恢复后未再次触发大面积超时

## 复盘关注点
1. 是否需要多供应商动态流量调度
2. 熔断与重试策略是否按供应商差异化配置
3. 是否对热门航线建立单独的兜底缓存方案
