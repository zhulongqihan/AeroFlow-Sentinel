# 航班搜索报价延迟突增处置手册

## 风险事件名称
- 事件名: `FlightSearchLatencySpike`
- 级别: 高
- 触发条件: 航班搜索报价链路 P99 连续 5 分钟高于 6 秒

## 业务影响
- 用户搜索结果返回明显变慢，首屏转化率下降
- 热门航线报价获取失败，出现空结果或价格刷新不一致
- 搜索到预订的转化漏斗明显收窄
- 客诉集中在“搜不到票”或“价格变化过快”

## 常见根因
1. GDS/供应商接口超时或限速触发
2. search-aggregator 并发过高，报价聚合线程池耗尽
3. 报价缓存失效导致实时询价回源比例激增
4. pricing-engine 的税费和附加费计算变慢

## 建议排查路径
1. 查询 Prometheus 风险事件，确认延迟开始时间、持续时间和受影响链路
2. 查询 gds-gateway 日志，确认是否存在供应商超时、502/504 或缓存命中率下降
3. 查询 booking-transaction 日志，确认是否已经影响后续 PNR 创建
4. 检查最近是否发布了新的报价规则、舱位映射配置或搜索限流策略

## 即时处置建议
1. 启用报价缓存兜底，降低实时回源比例
2. 暂时下线高成本组合报价或复杂筛选条件
3. 将流量切换到备用供应商或降低单供应商并发
4. 对热门航线做静态化价格兜底和结果降级展示

## 恢复判定
1. 搜索链路 P99 恢复到 3 秒以内
2. GDS 超时率恢复到基线水平
3. 搜索结果空白率和报价失败率不再连续升高

## 复盘关注点
1. 是否建立供应商级别的容量模型和限流策略
2. 报价缓存和实时询价比例是否可动态调整
3. 搜索链路是否需要进一步拆分热点航线与长尾航线流量
